void ContributionLoader::deploy ( const Trentino::RuntimeModel::ContributionPtr &  contribution)

Creates the RuntimeModel object graph and performs wiring.

After deployment the contribution is in RuntimeModel::Contribution::Deployed state. During deployment all the RuntimeModel objects are constructed and wiring is performed. Component implementation instances are not created during deployment. The run() method is responsible for creating instances for ="true" implementations.

Parameters:
[in]contributionA shared pointer to a contribution that must be in RuntimeModel::Contribution::Installed state.
Exceptions:
std::invalid_argumentIf the passed contribution is not in RuntimeModel::Contribution::Installed state.

void ContributionLoader::unload ( const Trentino::RuntimeModel::ContributionPtr &  contribution)

Currently does nothing and is reserved for future use.

All the necessary unloading actions will take place during destruction of the contribution object that will be triggered as soon as the Runtime erases the shared pointer to the contribution. This method is reserved to perform possible additional unloading actions (that are not covered by the contributions destruction process) in future.

Parameters:
[in]contributionA shared pointer to a contribution that must be in RuntimeModel::Contribution::Installed state.
Exceptions:
std::invalid_argumentIf the passed contribution is not in RuntimeModel::Contribution::Installed state.
